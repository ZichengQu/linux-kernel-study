# 内核编译与安装手册

## 1. 准备工作

安装必要工具：

| 发行版                   | 命令                                                                                                |
| --------------------- | ------------------------------------------------------------------------------------------------- |
| RHEL/CentOS/OpenEuler | `yum install -y make gcc gcc-c++ flex bison ncurses-devel openssl-devel elfutils-libelf-devel bc dwarves` |
| Debian/Ubuntu         | `apt-get install -y build-essential flex bison libncurses-dev libssl-dev libelf-dev bc dwarves`           |
| clang | `sudo apt install -y clang llvm llvm-dev libelf-dev zlib1g-dev make gcc pkg-config` |
| x86下编译arm64依赖 | sudo apt-get update && sudo apt-get install -y gcc-aarch64-linux-gnu binutils-aarch64-linux-gnu rpm lld bc flex bison libncurses-dev libssl-dev libelf-dev dwarves rsync cpio fakeroot |
| 备注：可能还缺少少数依赖，待后续整理 | |
---

## 2. 清理历史编译

不同清理程度：

| 命令               | 说明                    |
| ---------------- | --------------------- |
| `make clean`     | 删除中间文件，保留 `.config`   |
| `make mrproper`  | 删除 `.config` 和大部分生成文件 |
| `make distclean` | 最彻底，恢复接近源码初始状态        |

加强版清理，适合彻底重来：

```bash
make distclean -s && make clean -s && make mrproper -s
```

注意：`mrproper` 和 `distclean` 会删除 `.config`，执行前确认当前配置已经备份。

## 3. 配置内核

生成或修改 `.config`：

| 命令                                   | 说明            |
| ------------------------------------ | ------------- |
| `find . -name "openeuler_defconfig"` | 查找默认配置文件      |
| `make defconfig` 或 `make openeuler_defconfig`           | 使用厂商默认配置      |
| `make menuconfig`                    | 交互式修改配置       |
| `make olddefconfig`                  | 更新旧配置，自动接受默认值 |

## 4. 编译与打包

编译产物：x86 → `bzImage`，arm64 → `Image`。

| 命令                           | 说明      |
| ---------------------------- | ------- |
| `make -j$(nproc)`            | 编译内核    |
| `make binrpm-pkg -j$(nproc)` | 打包成 rpm |

常用普通编译流程：

```bash
vim Makefile && \
make clean -s && \
make openeuler_defconfig -s && \
make menuconfig -s && \
make -j$(nproc) -s
```

常用 rpm 包编译流程：

```bash
make clean -s && \
make openeuler_defconfig -s && \
make menuconfig -s && \
make binrpm-pkg -j$(nproc) -s
```

---

## 5. 安装内核

安装方式：

源码编译后换内核：

```bash
sudo make modules_install -j$(nproc) -s && \
sudo make install -j$(nproc) -s && \
sudo grub2-set-default 0
```

rpm 包强制安装：

```bash
sudo rpm -ivh --force ./*.rpm
```

## 6. GRUB 操作

管理启动项：

| 命令                                                                                | 说明           |
| --------------------------------------------------------------------------------- | ------------ |
| `grubby --info=ALL`                                                               | 查看所有内核信息     |
| `awk -F\' '$1=="menuentry " {print i++ " : " $2}' /boot/grub2/grub.cfg`           | 查看内核列表        |
| `grub2-set-default <index>`                                                       | 设置指定序号内核为默认启动 |
| `grubby --set-default /boot/vmlinuz-<ver>`                                        | 设置指定内核为默认启动  |
| `sudo grub2-mkconfig -o /boot/grub2/grub.cfg`                                     | 更新 grub 配置   |

查看所有内核，包含 `grubby` 信息和 openEuler UEFI 启动项：

```bash
grubby --info=ALL && \
sudo awk -F \' '$1=="menuentry " {print i++ " : " $2}' /boot/efi/EFI/euleros/grub.cfg
```

查看所有内核，简洁版：

```bash
sudo awk -F \' '$1=="menuentry " {print i++ " : " $2}' /boot/grub/grub.cfg
```

设置某个序号的内核为默认启动：

```bash
sudo grub2-set-default <index>
```

## 10. 修改内核启动参数

**场景**：切换 cgroup v1/v2，或添加调试参数。

### 方法一（推荐，持久生效）

1. 编辑 `/etc/default/grub`：

   ```bash
   sudo vim /etc/default/grub
   ```

   在 `GRUB_CMDLINE_LINUX` 中追加参数，例如：

   ```bash
   GRUB_CMDLINE_LINUX="quiet splash systemd.unified_cgroup_hierarchy=1 cgroup_no_v1=all"
   ```
2. 更新 grub 配置：

   * Debian/Ubuntu:

     ```bash
     sudo update-grub
     ```
   * RHEL/CentOS/OpenEuler (BIOS):

     ```bash
     sudo grub2-mkconfig -o /boot/grub2/grub.cfg
     ```
   * RHEL/CentOS/OpenEuler (UEFI 示例):

     ```bash
     sudo grub2-mkconfig -o /boot/efi/EFI/centos/grub.cfg
     ```
3. 重启：

   ```bash
   sudo reboot
   ```

### 方法二（临时/快速，不推荐长期）

直接修改 grub 配置文件：

```bash
sudo vim /etc/grub2-efi.cfg
```
```bash
sync
```
```bash
reboot
```

在对应内核的 `linux` 行添加参数 → 保存 → `sync` → `reboot`。
⚠️ 缺点：下次执行 `grub2-mkconfig` 会被覆盖。

## 11. 换核后调试/应急命令

打印所有任务栈：

```bash
echo 1 | sudo tee /proc/sys/kernel/sysrq
echo t | sudo tee /proc/sysrq-trigger
dmesg -T | tee qzc_dmesg.log
```

主动触发内核崩溃：

```bash
echo c | sudo tee /proc/sysrq-trigger
```

## 12. 一键流程示例

快速替换并重启（谨慎用）：

```bash
make clean -s && \
make openeuler_defconfig -s && \
NR_CPUS=$(nproc) && \
make -j$NR_CPUS -s && \
make modules_install -j$NR_CPUS -s INSTALL_MOD_STRIP=1 && \
make install -j$NR_CPUS -s INSTALL_MOD_STRIP=1 && \
sudo grub2-set-default 0 && \
sudo reboot
```
